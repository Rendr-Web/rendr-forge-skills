# Repo conventions

Skills live flat under `skills/`, one folder per skill, each with a `SKILL.md`.

Every skill must:
- have a **Reference** entry in the top-level `README.md`, with the skill name linked to its `SKILL.md`
- have an entry in `.claude-plugin/plugin.json` (`skills` array)

Keep skill bodies under ~150 lines; push exhaustive material into `references/` and link to it from the `SKILL.md`. Shared vocabulary lives in `skills/deslop/GLOSSARY.md`. Use its terms (Hole / Hardening / Improvement; Suspected / Confirmed / Closed) verbatim across all skills; don't introduce synonyms.

The pipeline is **stack-agnostic**. No tech-vendor brand names in the public skills; describe stacks by archetype (see `skills/setup-deslop/references/MAPPING-PATTERN.md`). Stack specifics belong in a per-project `STACK.md` generated by `/setup-deslop`, never hardcoded into a skill.

## Decisions & boundaries

- **`docs/adr/`**: Architecture Decision Records for the repo's own design. Tiny: a title + 1–3 sentences. Only record a decision that is hard to reverse, surprising without context, and the result of a real trade-off. Number sequentially (`0001-slug.md`).
- **`.out-of-scope/`**: one note per deliberately-rejected feature/request, with the reasoning. Preempts "why didn't you do X" issues. Append a `## Prior requests` link when a real request comes in.
- **`scripts/`**: `list-skills.sh` (enumerate skills) and `link-skills.sh` (symlink into `~/.claude/skills` for local dev).

Shared language lives in `skills/deslop/GLOSSARY.md`. It is the single source of vocabulary for the whole repo, so there is no separate root `CONTEXT.md`.
